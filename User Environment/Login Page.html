A Study on Non-Standard Input Devices and Interfaces

This project looks at the interfaces used for alphanumeric input with non-standard devices, such as TV remotes and gamepads as opposed to keyboards and touchscreens. The interfaces for these devices are often simple and wildly inefficient, and can be tedious or annoying for users to use with any significant frequency. This also hampers the design space of programs that utilise these devices. The aim of this project is to gather data about a range of input interfaces that are applicable to a range of input situations.

Currently, it seems that developers rarely implement anything beyond basic interfaces as there is no guiding standard or research. The end goal of this project is to be able to produce a system that is able to provide information that can be used to quantitatively decide upon the best input method for a developer to implement given a number of factors. These factors are the type and frequency of input(s), perhaps with extension to include the demographics of their users and a number of other factors. It is anticipated that developers may use this resource to implement suitable interfaces, increasing efficiency and reducing user error rates as a result.
